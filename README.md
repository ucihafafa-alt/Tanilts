# ТАНИЛ — Админ зуучлалтай танилцах систем v1.0

Энэ хувилбар нь хэрэглэгч хоорондын шууд чат, гишүүдийн нээлттэй жагсаалтгүй. Админ хүсэлтийг хянаж, тохирох хоёр хүнд нэргүй ерөнхий мэдээлэл санал болгоно. Хоёр тал зөвшөөрсний дараа админ холбоо барих мэдээллийг нээнэ.

## Системд орсон боломжууд

- 18+ үнэгүй бүртгэл
- Утас + 6 оронтой PIN нэвтрэлт
- Галерейгаас зураг сонгохгүй, камераар тухайн үед селфи авах
- Гэр бүл, тогтвортой үерхэл, нөхөрлөл, 18+ харилцан зөвшөөрсөн дотно харилцааны зорилго
- Хайж буй хүний нас, хүйс, байршил, хүүхэд, өмнөх гэрлэлт, тамхи, согтууруулах ундаа, нүүх боломж зэрэг шалгуур
- Энгийн хүсэлт: 7–14 хоног
- Шуурхай хүсэлт: 24–72 цагийн давуу хайлт, төлбөртэй
- Эхний 100 баталгаажсан энгийн хүсэлтийн урамшууллын тоолуур
- Хүсэлт өөрчлөх, цуцлах
- Админы мэдэгдэл
- Админ/операторын support чат
- Хоёр тал зөвшөөрсний дараа холбоо барих мэдээлэл нээх
- Бодит баталгаажсан гишүүний тоо

## Чухал

Систем 500–800 гэсэн зохиомол гишүүний тоо харуулахгүй. `Нийт бүртгэл`, `Баталгаажсан гишүүн`, `Идэвхтэй хүсэлт` нь Firestore дахь бодит тоогоор админ самбараас автоматаар шинэчлэгдэнэ.

## GitHub Pages дээр байрлуулах

1. Энэ ZIP-ийн бүх файлыг задлана.
2. GitHub-ийн `Tanilts` repository-ийн үндсэн хэсэгт бүх файлыг upload хийнэ.
3. Ижил нэртэй файлуудыг Replace хийнэ.
4. `Commit changes` дарна.
5. 1–3 минут хүлээнэ.

`https://ucihafafa-alt.github.io/Tanilts/`

Админ:

## Firebase тохиргоо

### 1. Authentication

Firebase Console → Authentication → Sign-in method → Email/Password-ийг Enable хийнэ.

### 2. Firestore Rules

1. Firebase Console → Firestore Database → Rules
2. Одоогийн кодыг бүхэлд нь устгана.
3. `FIRESTORE-RULES-PASTE.txt` файлын кодыг бүтнээр нь тавина.
4. `Publish` дарна.

GitHub-д `firestore.rules` upload хийх нь Firebase Rules-ийг автоматаар Publish хийхгүй.

### 3. Админ эрх

1. Админ болох утасны дугаараар хэрэглэгчийн сайтад нэг удаа бүртгүүлнэ.
2. Firebase Console → Authentication → Users дээр тухайн хэрэглэгчийн UID-г хуулна.
3. Firestore Data → `admins` collection үүсгэнэ.
4. Document ID = админы UID
5. `active` гэсэн boolean field = `true`
6. Админ хуудсаар нэвтэрнэ.

### 4. Үйлчилгээний тохиргоо

Админ нэвтэрсний дараа:

- Тохиргоо → энгийн үнэ
- шуурхай үнэ
- урамшууллын хязгаар
- банк, данс, хүлээн авагч

мэдээллийг хадгална. Админ анх нэвтрэх үед `appSettings/public` болон `publicStats/app` document автоматаар үүснэ.

## Шинэ урсгал

1. Хэрэглэгч бүртгүүлнэ.
2. Админ гишүүний селфи болон мэдээллийг хянаад “Батлах” дарна.
3. Хэрэглэгч энгийн эсвэл шуурхай хүсэлт илгээнэ.
4. Админ хүсэлтийг зөвшөөрч, хайлт эхлүүлнэ.
5. Админ “Танилцуулга” хэсгээс хоёр баталгаажсан хэрэглэгч сонгоно.
6. Хоёр хэрэглэгч нэргүй ерөнхий мэдээлэл харж зөвшөөрөх/татгалзах сонголт хийнэ.
7. Хоёулаа зөвшөөрвөл админ “Холбоо нээх” дарна.
8. Нэр, утас хоёр талд зэрэг нээгдэнэ.

## Хуучин тест бүртгэл

Өмнөх хувилбарын `users` document-ууд шинэ бүртгэлийн бүх талбаргүй байж болно. Шинэ системийг шалгахдаа шинэ утасны дугаараар бүртгүүлэх нь хамгийн найдвартай. Хуучин тест бүртгэлийг ашиглах бол Firestore болон Authentication дахь хуучин бүртгэлийг цэвэрлээд дахин бүртгэнэ.

## Нийтийн хэрэглээнд гаргахын өмнө

- Firebase App Check идэвхжүүлэх
- Cloud Functions/Admin SDK ашиглан статистик, урамшууллын эрх, бүртгэл устгалтыг сервер талаас баталгаажуулах
- Нууцлалын бодлого, үйлчилгээний нөхцөлөө хуульчтай нягтлуулах
- Селфи болон эмзэг мэдээллийн хадгалалтын хугацааг тогтоох
- Хэрэглэгчийн Auth бүртгэл устгах ажиллагааг Admin SDK-ээр хийх
- Firestore Rules Playground болон Emulator Suite-ээр дүрмээ турших

Энэ багц нь ажиллах прототип бөгөөд өндөр ачаалалтай нийтийн үйлдвэрлэлийн системд шууд ашиглахаас өмнө сервер талын баталгаажуулалт нэмэх шаардлагатай.
